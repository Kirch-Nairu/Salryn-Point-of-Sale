# Backup and Restore

Salryn POS supports local backup and restore workflows for store safety.

## Why Backups Matter

Store data may include:

- products
- inventory records
- sales history
- receipts
- audit records
- settings
- license state

Backups help protect store records from accidental deletion, device failure, or setup mistakes.

## Basic Backup Practice

Recommended store habit:

- create regular backups
- copy backups to a separate drive or trusted storage location
- do not store the only backup on the same failing computer
- verify important backups before major changes

## Data Folder Notice

Do not delete the Salryn data folder unless instructed during support.

It may contain important runtime files such as:

- local database
- receipts
- backups
- exports
- logs
- configuration
- license state
